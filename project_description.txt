========================================================================
                     MYNZO - PROJECT DESCRIPTION
========================================================================

Mynzo is a modern, premium e-commerce and interactive gaming marketplace platform. 
The workspace is structured into three main modules:
1. Frontend (User Client Application)
2. Admin-Frontend (Administrative Panel)
3. Backend (API Services and Integrations)

------------------------------------------------------------------------
TECHNICAL STACK
------------------------------------------------------------------------
- Backend: Node.js, Express, MongoDB (Mongoose ODM), Socket.io, Firebase Admin SDK
- Frontend & Admin: React, Vite, Vanilla CSS / Tailwind
- Database: MongoDB Atlas cloud cluster

------------------------------------------------------------------------
CORE INTEGRATIONS & SYSTEMS
------------------------------------------------------------------------
1. SMS Integration (SMS India Hub):
   - Fully DLT compliant.
   - Leverages DLT parameters (EntityId / dlttemplateid) for strict TRAI compliance.
   - Configured with Mynzo Sender ID (IIDMTB) and associated template mappings.

2. Shipping & Tracking (Shiprocket):
   - Integrates Shiprocket webhooks to receive order tracking updates.
   - Automatically dispatches SMS alerts to customers upon status changes.

3. Push Notifications (Firebase Admin SDK):
   - Handles Firebase Cloud Messaging (FCM) to send notifications.
   - Credentials configured securely as a parsed single-line JSON environment variable.

------------------------------------------------------------------------
CUSTOM CONFIGURATIONS & OPTIMIZATIONS
------------------------------------------------------------------------
- DB-first Admin Management:
  The backend checks for existing admin users in the database first. It avoids 
  crashing or requiring `ADMIN_EMAIL`/`ADMIN_PASSWORD` in the `.env` file if 
  admin users are already present.

- Environment Variables Format:
  `FIREBASE_SERVICE_ACCOUNT` is optimized into a single-line JSON format in `.env` 
  to prevent parsing errors caused by standard dotenv multi-line processing limits.

- Production Test Phone Bypass:
  Supports bypass of SMS sending for numbers listed in `TEST_PHONE_NUMBERS` 
  (e.g., `6268204871`). These test accounts automatically generate the static 
  OTP `123456` in all environments (including production) without sending actual SMS.
